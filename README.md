# NeuroMultiAgentSystem

**多智能体深度检索医疗辅助系统 (Multi-Agent Deep Retrieval Model)**

## 一、项目定位与总体简介

本项目是一个面向**脑卒中临床场景**的智能医疗辅助系统，旨在通过人工智能技术提升临床辅助决策支持能力。系统以大语言模型为核心，融合检索增强生成（RAG）、多智能体推理与医学知识库，实现了从症状输入到辅助分析与建议输出的完整闭环。

与传统通用问答系统不同，本项目并非简单依赖模型生成结果，而是以权威医学文献与临床指南为知识底座，通过“检索—重排—推理—评估”的结构化流式推理，确保生成的每一条回答都具备明确的依据来源与可靠性。系统引入“证据先行”理念，在保证模型高水平的同位时，显著降低了医学场景下的可能性与安全隐患。

在应用定位上，本项目面向医生、医学生及医疗相关从业人员，同时兼顾患者健康管理需求：
- **医生与医学生**：系统可辅助医生快速查阅脑卒中相关指南、分析病例并提供参考建议；也可用于医学教学与学习场景，帮助理解疾病机制与诊疗思路。
- **患者健康管理**：支持个体化健康数据的整合与分析，使AI能力从单次问答扩展到连续医疗服务过程。

本项目不仅是一个AI模型应用，更是一个融合前端交互、后端业务逻辑与模型推理服务的完整工程系统，为构建面向真实医疗场景的临床决策支持系统（CDSS）提供了可行的技术路径。

## 二、系统整体功能

本项目围绕真实医疗使用场景，构建了“智能问诊分析—患者管理—医学知识学习”三大核心功能模块。

### 1. 智能问诊与AI辅助分析模块
该模块是系统的核心功能入口。系统接收症状描述后，通过医学问题理解与任务拆解，从医学文献库中检索证据，结合多智能体推理生成结构化分析结果。
- **结构化输出**：包括最可能诊断及依据、解剖定位分析、病理机制解释、置信度评估及需排除的重要诊断等，贴近真实临床思维。
- **流式实时反馈**：用户可实时看到AI推理过程与阶段性结果，显著提升交互体验与信任感。

### 2. 患者档案与个体化分析模块 (EHR)
系统引入患者管理模块，记录患者基本信息、既往病史及医生备注。
- **持续性健康管理**：结合历史记录与当前输入进行综合分析，实现个体化风险评估与辅助诊断建议，模拟真实临床连续诊疗过程。

### 3. 医学知识学习与文献检索模块
系统内置多篇国内权威医学指南（如脑卒中相关指南）并提供 PubMed 支持。
- **文档支持与检索增强**：不仅提供在线阅读与结构化浏览，这些文献还作为 RAG 核心知识来源参与 AI 推理，确保结果建立在真实医学资料之上。

## 三、系统架构与前后端设计

本项目采用典型的前后端分离与模型服务独立部署的三层架构：

### 1. 前端交互层 (Frontend)
- **技术栈**: Vue 3 (Composition API) + Vite 7 + Pinia + SCSS
- **核心设计**: 以用户体验为核心，通过 `fetch` + `ReadableStream` 持续接收后端推送，实现流式数据实时渲染。支持医学文档在线预览（PDF）、图片上传（多模态扩展）以及结构化结果展示。

### 2. 后端服务层 (Backend / MyServer)
- **技术栈**: Java 17 + Spring Boot 3 + Spring WebFlux + MySQL 8.0 + Redis 6.0 + Redisson
- **核心设计**: 采用响应式编程模型（WebFlux）支持高并发，通过 JWT 实现身份认证与安全控制。Redisson 控制分布式并发，基于 ThreadLocal 隔离请求。对于流式推理任务，后端通过 `WebClient` 对模型服务进行流式调用并转发至前端。

### 3. 模型推理服务层 (Model)
- **技术栈**: Python 3.11 + FastAPI + 异步 asyncio
- **核心设计**: 统一入口加载大语言模型（Qwen系列）、混合检索引擎（Hybrid RAG）、重排模型（Reranker）以及多智能体推理模块。通过异步生成器持续输出 `thinking`、`chunk`、`done` 等标准事件格式，实现流式通信。

### 🔄 全链路流式数据管道 (SSE Pipeline)
```text
用户提问 ➡️ Java鉴权与限流 ➡️ WebClient ➡️ FastAPI ➡️ Python模型(yield) ➡️ asyncio.Queue ➡️ Java(Flux推送) ➡️ Vue(ReadableStream实时渲染)
```

## 四、核心技术与模型能力

### 1. 深度定制的混合检索与高级 QA 重排引擎 (Advanced RAG & Rerank)
系统突破了传统大模型“直接回答”的局限，实现了“基于证据组织答案”的严谨医疗推理范式：
- **常规底层知识库建设**：以句为分割，正常切割
- **高质量底层知识库建设**：在向量库构建阶段引入批处理扩写技术，通过系统精读医疗 PDF 文档，自动提炼并生成高质量的 QA（问答）对，夯实知识底座
- **多路召回与混合检索**：采用向量检索与 BM25 结合的混合策略，既保证了自然语言的语义相似性，又兼顾了严谨医学术语的精确匹配。
- **指南优先与重排优化**：针对脑卒中等专科临床场景，设计了“指南优先检索策略”，优先召回权威临床指南与最新文献。初步召回后，结合阿里 `gte-rerank` 模型进行深度语境打分与证据压缩，剔除冗余信息。
- **高可信度溯源**：在最终生成的诊断与建议结果中，对引用的证据进行明确的文献名称与页码溯源，确保每一条医疗推理都有据可查，大幅降低大模型幻觉风险。

### 2. 多智能体协同推理机制 (Proposer-Critic-Integrator)
创新引入 LangGraph 多智能体协同推理机制，将临床推理拆分为三级角色：
- **Proposer (生成智能体)**：基于证据生成初步诊疗方案（鉴别诊断、TOAST分型、时间窗决策等）。
- **Critic (审查智能体)**：独立执行风险审查，识别6大类临床高风险点（如时间窗陷阱、禁忌症、致命遗漏）。
- **Integrator (整合反思智能体)**：对初步方案与审查意见进行融合反思，生成最终安全结论。

### 3. 流式推理与多模态分析
采用流式生成（Streaming）机制，包含可见的思考过程（Thinking Step）。系统并预留多模态扩展接口（如 Qwen-VL），可对医学影像或报告图片进行识别与解析。

## 五、创新点与作品亮点

1. **证据驱动的医学推理范式创新**：将证据获取前置，实现了从“模型主导”向“证据主导”的转变，降低幻觉风险。
2. **多智能体协同的安全推理机制**：“生成-审查-反思”的三级架构模拟了临床“提出方案→上级把关→最终复盘”的真实流程，提升系统鲁棒性与安全性。
3. **面向脑卒中场景的深度定制 RAG**：强化对时间窗、溶栓/取栓指征、推荐等级等关键信息的提取，从通用AI走向专科AI。
4. **极速与安全双路径设计**：简单场景极速响应，高风险问题自动切换至多智能体深度分析模式。
5. **诊疗、管理与学习的闭环整合**：将智能问诊、患者管理与知识学习有机融合，支持长期随访与动态优化。
6. **工程化与可扩展系统设计**：三层解耦架构，配置化的提示词与模型调用逻辑，可极低成本迁移至其他病种。

## 六、应用价值与落地前景

- **临床辅助决策支持**：脑卒中等高时效性疾病中，帮助医生快速定位时间窗、适应证等要点，提高决策效率。
- **医学教学与培训**：适用于住培及医学教育，通过指南与结构化推理帮助学习者理解临床思维。
- **患者管理与健康服务**：记录个体化数据，支持慢病管理、康复跟踪及长期随访。
- **医学科研支持**：辅助科研人员快速定位文献脉络与核心结论。
- **CDSS 原型基础**：具备临床决策支持系统的核心组件，未来可直接接入 HIS/PACS 系统。

## 💻 开发者指南

1. **环境要求**：
   - MySQL 8.0+ / Redis 6.0+ / JDK 17+
   - Node.js `^20.19.0 || >=22.12.0`
   - Python 3.11+
2. **代码规范与兼容性**：
   - Java遵循 WebFlux 响应式范式，Python全程使用 `async/await`，前端遵循 Vue 3 Composition API。所有通信必须兼容现有的 SSE 数据结构。
3. **本地启动**：
   - **后端**: 修改 `backend/src/main/resources/application.yml` 中的 MySQL 和 Redis 连接，运行 `MyServerApplication.java`。
   - **前端**: 进入 `frontend/` 目录执行 `npm install` 与 `npm run dev`，默认代理至 `8080` 端口。
