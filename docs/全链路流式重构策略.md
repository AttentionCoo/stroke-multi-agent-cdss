# Synapse MD 全链路流式重构策略

**项目**：Synapse MD · Python 后端 + Java 中间件 + Vue 前端  
**制定日期**：2026-04-01  
**状态**：待执行  
**前置分析**：`LangChain迁移可行性分析报告.md`、`LangChain版本升级风险分析报告.md`、`streaming-analysis-report.md`

---

## 一、目标

在完成架构复原（Python 推理管线回归原始四节点结构）的同时，对三层链路的流式通路做一次彻底重设计，消除以下现存问题：

| 现存问题 | 根因 |
|----------|------|
| SSE 心跳机制脆弱（依赖 `asyncio.wait` + `__anext__` 取消行为） | 无标准保活机制，靠手工 hack |
| LangChain 版本严重滞后（0.1.20），与 LangGraph 不兼容 | 长期未升级 |
| `qwen_agent.py` 死代码堆积（MCQ 路径、未使用的 `_build_graph`） | 历次迭代遗留 |
| 前端 Markdown 渲染闪烁 | 逐 token 触发 Vue 响应式更新 |
| Nginx 代理缓冲可能吞掉 SSE 事件 | `proxy_buffering` 默认开启 |

---

## 二、整体架构

```
Python FastAPI
  LangGraph StateGraph（四节点原始管线）
    └─ astream_events(version="v2")
      └─ 事件翻译层（LangGraph 内部事件 → 业务 SSE 事件格式）
        └─ sse-starlette EventSourceResponse（ping_interval=15，替代 asyncio.wait 心跳）

                ↓  text/event-stream（标准 SSE 协议）

Spring WebFlux
  WebClient.bodyToFlux(ServerSentEvent.class)
    └─ 过滤 SSE ping 帧（协议层心跳不透传前端）
    └─ limitRate() 背压控制
    └─ timeout(Duration.ofMinutes(5))
    └─ 以相同 SSE 格式重新输出给前端

                ↓  text/event-stream

Vue 3
  fetch + ReadableStream（非 EventSource，支持 POST 和 Auth header）
    └─ pendingBuffer + findSafeBoundary（段落级渲染，消除 Markdown 闪烁）
    └─ requestAnimationFrame 批量调度 DOM 更新
```

## 三、SSE 事件格式契约（三端共同遵守）

以下为 Python → Java → Vue 链路的统一事件格式，**在编码前必须三端对齐确认**。

### 3.1 事件类型定义

| type | 触发时机 | 关键字段 |
|------|----------|----------|
| `node_start` | LangGraph 节点开始执行 | `node`（节点名）、`label`（中文提示） |
| `node_done` | LangGraph 节点执行完毕 | `node`、`summary`（可选摘要） |
| `token` | LLM 流式输出每个 token | `content` |
| `done` | 流正常结束 | `request_id`、`name`（会话名）、`all_info`（上下文摘要） |
| `error` | 任意层级发生错误 | `code`（结构化错误码）、`message` |

> SSE 协议层心跳（`sse-starlette` 自动发出的空注释行 `: ping`）由 Spring WebFlux 层过滤，**不作为业务事件**，前端从不感知。

### 3.2 事件示例

```jsonc
// 节点开始
{"type": "node_start", "node": "intent",    "label": "正在判断问题类型..."}
{"type": "node_start", "node": "analysis",  "label": "正在分析病例结构..."}
{"type": "node_start", "node": "retrieve",  "label": "正在检索循证医学证据..."}
{"type": "node_start", "node": "reason",    "label": "正在进行临床推理..."}

// 节点结束
{"type": "node_done",  "node": "analysis",  "summary": "提取到 3 个临床问题"}
{"type": "node_done",  "node": "retrieve",  "summary": "检索到 5 个证据片段"}

// 流式 token
{"type": "token", "content": "根据患者病史"}
{"type": "token", "content": "，建议..."}

// 正常结束
{
  "type": "done",
  "request_id": "a1b2c3d4e5f6",
  "name": "脑卒中急性期处理咨询",
  "all_info": "患者男，62岁，右侧肢体无力2小时..."
}

// 错误
{"type": "error", "code": "ERR_RETRIEVE_FAILED", "message": "向量库检索超时"}
```

### 3.3 废弃的旧事件类型

| 旧 type | 废弃原因 | 替代方案 |
|---------|----------|----------|
| `thinking` | 语义模糊，字段不统一 | `node_start` + `node_done` |
| `chunk` | 名称不清晰 | `token` |
| `result` | 与 MCQ 路径强绑定（MCQ 路径已删除） | `token` + `done` |
| `meta` | 拆分为 `done` 携带元数据 | `done.all_info` + `done.name` |
| `heartbeat` | 由 SSE 协议层接管 | sse-starlette `ping_interval` |

**`Agent/qwen/qwen_agent.py`（完全重写）**

目标：删除全部死代码，以 LangGraph StateGraph 实现原始四节点管线。

```
原始管线结构：

   ↓
[route]    ── 条件路由

   ├── "knowledge"    → LLM 直接流式回答，END
```

### 4.2 Spring WebFlux 层

- `WebClient` 消费端加 `.filter(e -> !"ping".equals(e.event()))` 过滤 SSE 协议层心跳
- 按新事件格式（`node_start/node_done/token/done/error`）调整事件处理逻辑
- 删除对旧事件类型（`thinking/chunk/result/meta/heartbeat`）的处理
- 显式设置 `.timeout(Duration.ofMinutes(5))`，不依赖 Nginx 超时
- 加 `.limitRate(32)` 背压控制

---

### 4.3 Vue 3 层

- 按新事件格式调整事件处理逻辑
- 新增 `useStreamRenderer` composable，实现：
  - `pendingBuffer` 累积 token
  - `findSafeBoundary` 检测段落安全边界（空行且不在代码块内）
  - `requestAnimationFrame` 批量调度 DOM 更新
  - 流结束时 `flush` 剩余内容
- 用 `node_start/node_done` 驱动进度指示器（替代旧的 `thinking` 事件）
- `done` 事件携带 `all_info` 和 `name`，替代旧的 `meta` 事件

## 七、决策记录

| 决策 | 选择 | 放弃的选项 | 理由 |
|------|------|-----------|------|
| 流式协议 | SSE | WebSocket | 请求-响应模型，无双向通信需求 |
| 事件翻译位置 | Python FastAPI 层 | Java WebFlux 层 | Java 不应感知 LangGraph 内部结构 |
| 心跳机制 | sse-starlette 协议层 ping | asyncio.wait 业务层心跳 | 标准化，不依赖生成器取消行为 |
| TokenAggregator | 废弃 | 保留 | Vue pendingBuffer 在更合适的层解决同一问题 |
| LangGraph Checkpointing | 不引入 | 引入 | 当前无断线续传需求，避免过早复杂化 |
| MCQ 路径 | 删除 | 保留 | 与医疗问诊主场景无关，属历史遗留 |
| vision_service.py | 不动 | 迁移到 LangChain | dashscope VL 流式参数无 LangChain 等价实现 |
